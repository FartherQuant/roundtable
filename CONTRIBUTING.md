# 贡献指南

感谢你对 Roundtable 项目的兴趣！最简单的贡献方式是提交新的人格配方。

## 提交人格配方

### 1. 创建 YAML 文件

在 `personas/` 目录下创建新文件，命名为 `{英文名}.yaml`，例如 `hayek.yaml`。

### 2. 填写4个要素

参考 [persona-schema.yaml](persona-schema.yaml)，必须包含：

```yaml
id: persona-hayek
name: 哈耶克
version: v1.0

persona:
  speaking_style: "一句话描述说话风格"
  stance_anchors:
    - "追问方向1——动词开头"
    - "追问方向2——动词开头"
  honesty_rules: "什么情况下承认自己的方法不适用"
  anti_pattern: "核心方法过度延伸时会犯什么错"
```

### 3. 3条自检（必须回答）

提交前回答这3个问题，写在 PR 描述中：

1. **为什么选这个人？** 这个人为什么比其他同类人物更适合圆桌讨论？
2. **负责什么视角？** 他在圆桌讨论里负责推进哪个追问方向？
3. **拿掉缺什么？** 把他拿掉之后，会缺哪一个关键视角？

3条说不清 = 这个人格可能不适合加入。

### 4. 提交 PR

- 文件放到 `personas/` 目录
- PR 标题格式：`[persona] 添加 {人格名}`
- PR 描述包含3条自检回答

## 提交 prompt 模板

如果你设计了新的讨论模式（如辩论模式、法庭模式），欢迎提交到 `templates/` 目录。

- 文件命名：`{模式名}.md`
- PR 标题格式：`[template] 添加 {模式名}`

## 设计原则

1. **极简** — 每个人格4个要素，不多不少
2. **可组合** — 任何3-5个人格组合都能产生有效讨论
3. **有边界** — 每个人格必须声明自己不适用的情况（honesty_rules）
4. **防过度延伸** — 每个人格必须声明自己的反套路（anti_pattern）
